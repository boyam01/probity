# Internationalization (i18n) Plan

This is a **plan**, not an implementation. **No i18n code ships in this pass.**

## Principles

- **English remains primary** for public docs.
- **zh-TW** gets: the README summary, the [Quickstart](QUICKSTART.md), and report
  explanations.
- **Verdict enum names stay stable in English:** `KILL` / `PASS` / `INSUFFICIENT`
  and the reason codes (`ENV_UNSTABLE`, `AUDIT_INTEGRITY`, `LOW_POWER`,
  `CRITICAL_EVENT`, `RELIABILITY_REFUTED`, `CI_STRADDLES_THRESHOLD`).
- **Translate explanations, never schema keys or enum values.** Reports stay
  machine-readable in one canonical form; translation is a presentation layer.

## Proposed future CLI UX (NOT implemented)

```text
probity explain report.json --lang zh-TW
probity explain report.json --lang en
```

`explain` would render a human-readable explanation of an existing audit report
in the chosen language. It does not exist yet; today,
`python -m probity report <audit_json>` re-renders the English markdown.

## Terminology table (EN ↔ 繁體中文)

| English | 繁體中文 | Note |
|---|---|---|
| false green | 假綠燈 | a green / PASS signal not backed by evidence |
| claim | 宣稱 | the agent's own "done / tests pass" |
| evidence | 證據 | what the deterministic checker found |
| integrity gate | 完整性閘門 | the gate around claim ↔ evidence |
| canary | 金絲雀 (canary) | scripted pre/post environment-fault detector |
| provenance | 來源憑證 (provenance) | the `spec_hash` / `calibration_hash` chain |
| owner amendment | Owner 修訂 | a policy / provenance change the Owner alone authorizes |

## Scope boundary

Schema keys, enum values, file formats, and hashes are **not** translated;
prose explanations are. This keeps reports diff-stable and tooling
language-agnostic. Translating an enum value (e.g. `KILL`) would break the
machine-readable contract and is out of scope.
